SGE

Criando o App SGE (Sistema de Geolocalização Educacional)

Bem-vindos ao Projeto SGE! Neste módulo, vamos dar um salto no desenvolvimento Flutter: sairemos apenas da criação de telas visuais e vamos começar a controlar o hardware do celular (Câmera e GPS) e a manipular arquivos reais.

O nosso grande desafio de hoje surge de um problema do mundo real: quando você tira uma foto e envia pelo WhatsApp ou redes sociais, esses aplicativos "espremem" (comprimem) a imagem para ela ficar mais leve. Nesse processo, eles apagam todos os dados ocultos da foto (como a localização de onde ela foi tirada) por questões de privacidade.

Para resolver isso e garantir que o endereço chegue ao destino, nós vamos criar uma marca d'água à prova de compressão.

1. Acessando o Mundo Físico (Câmera e GPS)

Para o nosso aplicativo funcionar, precisamos de pacotes externos que conversem com as peças físicas do celular:

O image_picker: É a nossa ponte para a câmera. Com ele, ordenamos que o celular abra a câmera principal, tire uma foto e nos devolva um arquivo temporário na memória.

O geolocator: É o nosso acesso à antena GPS. Ele funciona mesmo sem internet, lendo os satélites para nos dar os números exatos de Latitude e Longitude (ex: Lat: -23.55, Long: -46.63).

2. O Motor de Busca e o Escudo do Try/Catch

Ter apenas os números da Latitude e Longitude não é muito amigável para ler. Por isso, usamos um terceiro pacote chamado geocoding, que pega esses números e vai até a internet procurar o nome da rua. Mas, e se o aluno estiver no meio do mato, sem 4G ou Wi-Fi? É aqui que entra a estrutura de segurança try/catch (Tentar / Capturar):

O aplicativo tenta (try) acessar a internet para descobrir a rua.

Se falhar (sem rede), ele não quebra o app. O erro cai na rede de segurança (catch), e nós programamos o app para ignorar a falha e usar apenas os números do GPS (Lat/Long), que foram capturados offline.

3. A Mágica dos Pixels (A Marca D'água)

A parte mais importante do aplicativo! No Flutter, colocar um widget Text por cima de um widget Image é como colar um adesivo na tela da sua TV. Se você enviar a TV para alguém, o adesivo não vai junto com a fiação de dentro dela.

Para a nossa marca d'água ser permanente, usamos o pacote image. Ele pega o arquivo da foto, transforma em uma matriz de milhões de pixels, e nós usamos o comando drawString() para pintar as letras da localização diretamente nos pixels da foto. Quando salvamos, o texto se funde à imagem para sempre!

💻 Guia Prático de Atividades Passo a Passo

Siga os roteiros estruturados abaixo para testar os recursos de hardware isoladamente no seu editor antes de montar o app SGE completo:

Atividade 1: O Rastreador Simples (Treinando o GPS)

Desafio: Criar uma tela contendo um botão centralizado. Ao clicar nele, o hardware do GPS deve ser ativado para capturar e exibir as coordenadas de Latitude e Longitude em tempo real na tela.

Passo 1: Instalação

Adicione a dependência correta no seu arquivo pubspec.yaml abaixo de dependencies:

dependencies:
  geolocator: ^13.0.0

Passo 2: Importação e Variáveis

No topo do seu arquivo main.dart, importe a biblioteca e defina a variável de texto que guardará os dados do GPS dentro do escopo do seu State:

import 'package:geolocator/geolocator.dart';

// Variável para armazenar o texto exibido na tela
String _textoGps = "Clique no botão para rastrear";

Passo 3: A Lógica de Captura

Crie o método assíncrono que solicita a permissão do usuário e faz a leitura física do satélite:

Future<void> _capturarCoordenadas() async {
  // Solicita permissão de uso do GPS ao sistema operacional do celular
  LocationPermission permissao = await Geolocator.requestPermission();
  
  if (permissao == LocationPermission.whileInUse || permissao == LocationPermission.always) {
    // Captura a posição geográfica atual do hardware
    Position posicao = await Geolocator.getCurrentPosition();
    
    // Atualiza o estado da tela para exibir os números
    setState(() {
      _textoGps = "Latitude: ${posicao.latitude}\nLongitude: ${posicao.longitude}";
    });
  }
}

Passo 4: Como testar

  • Abra o emulador ou conecte seu celular físico.
  • Clique no botão e **autorize o aplicativo** a acessar a localização.
  • Verifique se os números de Latitude e Longitude aparecen na interface.

Atividade 2: Simulador de Apagão (Tratamento com Try/Catch)

Desafio: Modificar a lógica anterior adicionando a busca pelo nome da rua. Caso o dispositivo seja desconectado da rede Wi-Fi/4G, o aplicativo deve capturar o erro de conexão de forma segura e continuar rodando.

Passo 1: Instalação

Adicione o pacote responsável por transformar números em nomes de ruas no seu pubspec.yaml:

dependencies:
  geocoding: ^3.0.0

Passo 2: Construindo o Escudo Try/Catch

Substitua a lógica de atualização incluindo a verificação online protegida por uma barreira antiqueda:

import 'package:geocoding/geocoding.dart';

Future<void> _buscarEndereco(double lat, double long) async {
  try {
    // Bloco TRY: O app tenta buscar os dados na internet
    List<Placemark> placemarks = await placemarkFromCoordinates(lat, long);
    Placemark local = placemarks.first;
    
    setState(() {
      _textoGps = "Rua: ${local.street}, ${local.subLocality}";
    });
  } catch (erro) {
    // Bloco CATCH: Executado automaticamente se o celular estiver offline
    setState(() {
      _textoGps = "Sem internet! Coordenadas mantidas: Lat $lat, Long $long";
    });
  }
}

Passo 4: Como testar

  • **Teste 1 (Online):** Clique no botão com a internet ligada. A interface deve exibir o nome da rua.
  • **Teste 2 (Offline):** Desative o Wi-Fi e os dados móveis do seu computador/emulador e clique novamente. O app não pode travar, ele deve exibir a mensagem amigável definida no `catch`.

Atividade 3: O Carimbo Digital (Pintando Pixels do Arquivo)

Desafio: Criar uma rotina em segundo plano capaz de abrir uma imagem da memória do celular, decodificar sua estrutura de bytes e rasterizar (desenhar) um texto definitivo por cima dela.

Passo 1: Instalação

Utilize a biblioteca pura de manipulação de dados de imagem no pubspec.yaml:

dependencies:
  image: ^4.3.0

Passo 2: Lógica de Escrita em Bytes

Implemente a função que transforma o arquivo físico em uma matriz de pontos coloridos para realizar a pintura definitiva:

import 'package:image/image.dart' as img;
import 'dart:io';

Future<void> _aplicarCarimbo(File arquivoFotoOriginal) async {
  // 1. Lê os bytes do arquivo e os converte em uma matriz de pixels manipulável
  final bytes = await arquivoFotoOriginal.readAsBytes();
  img.Image? imagemEmPixels = img.decodeImage(bytes);

  if (imagemEmPixels != null) {
    // 2. Desenha o carimbo de texto de forma definitiva diretamente na matriz
    img.drawString(
      imagemEmPixels,
      "SGE - COORDENADA SEGURA",
      font: img.arial24, // Define um tamanho de fonte legível
      x: 30,             // Afastamento da borda esquerda (Eixo X)
      y: 30,             // Afastamento do topo (Eixo Y)
    );

    // 3. Reconverte a matriz modificada de volta para o formato JPEG
    final bytesProcessados = img.encodeJpg(imagemEmPixels);
    
    // 4. Salva por cima do arquivo antigo, mesclando o texto de forma permanente
    await arquivoFotoOriginal.writeAsBytes(bytesProcessados);
  }
}

Passo 3: Como testar

  • Envie a foto gerada por essa função para outro dispositivo via chat de rede social.
  • Mesmo que a rede reduza a qualidade do arquivo, o texto permanecerá legível, provando que ele agora faz parte dos pixels da imagem.
